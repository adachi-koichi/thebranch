### ID: 2206 | [THEBRANCH] 組織ダッシュボード設計

**優先度**: 2 (中期)

#### ビジョン
組織全体の進捗を一目で把握。ひとりのマネジメント負荷を最小化。

#### ユーザーストーリー
組織のリーダーがダッシュボードで全体進捗・リスク・リソース利用状況を把握したい

#### 設計要件
1. 概観パネル（Overview）
   - 全体タスク数（pending / in_progress / done）
   - 部署別完了率（プログレスバー）
   - 本週完了数・目標達成度

2. リソース利用状況
   - アクティブエージェント数（リアルタイム）
   - 同時処理数制限の達成状況
   - リソース枯渇警告（CPU / メモリ）

3. タスクボード（Kanban 風）
   - 列: pending / in_progress / blocked / done
   - カードにはタイトル・優先度・担当部署
   - ドラッグ＆ドロップでステータス変更可能

4. リスク・ボトルネック検出
   - 滞留タスク（30分超：警告、60分超：緊急）
   - ブロッカー分析（どのタスクが何をブロックしているか）
   - エラー率・失敗タスクの可視化

5. アナリティクス
   - 部署別の完了率・平均処理時間
   - タスクタイプ別の内訳
   - 週別・月別のトレンド

6. ナビゲーション
   - 部署別ドリルダウン
   - タスク詳細表示・編集
   - メンバー別パフォーマンス

#### 技術要件
- ダッシュボード: React / Vue.js + D3.js / Chart.js
- リアルタイム更新: WebSocket / polling
- API: FastAPI エンドポイント（タスク・リソースデータ）
- データベース: タスク・メトリクスの集計クエリ最適化
