# オンボーディングフロー設計 — THEBRANCH

## コンセプト
「組織は生まれながらの特権ではない。アイデアを持つすべての個人が、翌朝から機能する組織を手にできる世界」

ユーザーが組織に参加して**最初の5分で**、組織の基本構造を理解し、最初のタスクを開始できるフロー。

---

## ユースケース

### UC-1: 新規メンバー参加時のセットアップ
- **主体**: 新規ユーザー（メンバー招待リンク経由でアクセス）
- **目標**: 組織コンテキストの把握 → ロール確認 → 最初のアクション決定
- **パス**: ウェルカム画面 → 組織概要 → ロール確認 → ダッシュボード遷移

### UC-2: 既存メンバーの再ログイン
- **主体**: 既存ユーザー（保存されたセッション or ログイン）
- **目標**: 直前のコンテキストを復帰させつつ、新規通知の確認
- **パス**: ログイン → ダッシュボード → 未読タスク表示

---

## 画面遷移フロー

```
[ウェルカム画面]
    ↓
  未登録 / 登録済み
  ├─ 未登録：メール認証
  └─ 登録済み：パスコード入力
    ↓
[組織コンテキスト選択]
  （複数組織メンバーの場合）
    ↓
[ロール確認・権限表示]
  （PM / EM / Engineer など）
    ↓
[初期セットアップ]
  - ユーザー名確認
  - プロフィール画像登録（オプション）
  - 通知設定（オプション）
    ↓
[ダッシュボード初期表示]
  - 割り当てられたタスク
  - チームメンバー一覧
  - 組織サマリー
```

---

## 各画面の定義

### 1. ウェルカム画面 (Welcome Screen)

**目的**: ユーザーの状態判定（新規 / 既存）

| 項目 | 内容 |
|---|---|
| **レイアウト** | フルスクリーン、シンプル背景 |
| **ビジュアル** | THEBRANCH ロゴ + スローガン |
| **テキスト** | 「あなたのアイデアが、翌朝から組織になる」 |
| **CTA ボタン** | 「ログイン」「新規登録」 |

**入力項目**: なし（ボタン選択のみ）

---

### 2. 認証画面 (Authentication)

#### 2a. 新規登録フロー
**タイトル**: メールアドレス認証

| 項目 | 内容 | 必須 | デフォルト |
|---|---|---|---|
| **メールアドレス** | テキスト入力 | ✓ | - |
| **認証ボタン** | 「認証コード送信」 | - | - |

**動作**: 
- メール送信 → 確認画面に遷移
- ユーザーがメール内のコードを入力 → 認証完了

#### 2b. 既存ユーザーログイン
**タイトル**: パスコード入力

| 項目 | 内容 | 必須 | デフォルト |
|---|---|---|---|
| **メールアドレス** | テキスト入力（省略可能）| - | 前回の記憶 |
| **パスコード** | 4～6 桁数字 | ✓ | - |
| **パスコード送信** | ボタン | - | - |

---

### 3. 組織コンテキスト選択画面 (Organization Context)

**条件**: ユーザーが複数の組織に属する場合に表示

| 項目 | 内容 |
|---|---|
| **タイトル** | 「どの組織で作業しますか？」 |
| **リスト** | 組織カード一覧（名前 + 説明 + メンバー数） |
| **選択動作** | カード選択 → ロール確認画面へ |
| **デフォルト** | 1組織のみ → スキップしてロール確認へ |

**表示情報**:
- 組織名
- 簡潔な説明（1行）
- メンバー数
- 自分のロール（PM / EM / Engineer など）

---

### 4. ロール確認・権限表示画面 (Role Confirmation)

**目的**: ユーザーの権限とアクションを明確化

| 項目 | 内容 |
|---|---|
| **タイトル** | 「あなたのロール: [ロール名]」 |
| **ロール説明** | PM / EM / Engineer などの定義と責務 |
| **権限表示** | 実行可能なアクション一覧 |
| **チームメンバー** | 自分が属するチームの簡潔な紹介 |
| **次へボタン** | 「初期セットアップへ進む」 |

**ロール別表示内容**:

| ロール | 説明 | 権限 |
|---|---|---|
| **PM** | プロジェクトマネージャー | タスク作成・割り当て・優先度設定 |
| **EM** | エンジニアリングマネージャー | チーム管理・進捗監視・品質確保 |
| **Engineer** | エンジニア | タスク実装・PR 作成・レビュー参加 |

---

### 5. 初期セットアップ画面 (Initial Setup)

**目的**: ユーザープロフィール情報の補完

| 項目 | 内容 | 必須 | 入力タイプ |
|---|---|---|---|
| **表示名** | テキスト入力 | ✓ | フリーテキスト |
| **プロフィール画像** | ファイル選択 | ✗ | 画像アップロード |
| **通知設定** | チェックボックス | ✗ | - |
| │ - メール通知を受け取る | チェック | ✗ | - |
| │ - Slack 連携 | チェック | ✗ | - |
| **セットアップ完了** | ボタン | - | - |

**動作**:
- 必須項目入力後 → 「セットアップ完了」ボタンが有効化
- 完了 → ダッシュボード初期表示へ

---

### 6. ダッシュボード初期表示 (Dashboard Entry Point)

**目的**: ユーザーが最初に見る組織のメインビュー

| セクション | 内容 |
|---|---|
| **ウェルカムメッセージ** | 「[ユーザー名]さん、ようこそ！」 + 今日のサマリー |
| **割り当てられたタスク** | 自分の担当タスク（進行中 TOP 3） |
| **チームメンバー** | 同じチームのメンバー一覧（オンライン状態表示） |
| **クイックアクション** | ロール別のすぐやることリスト |
| **組織サマリー** | 全体進捗・アクティブなプロジェクト数 |

**ロール別クイックアクション**:
- **PM**: 「タスク作成」「チーム一覧を見る」
- **EM**: 「チーム進捗確認」「メンバー管理」
- **Engineer**: 「割り当てタスク確認」「PR 作成」

---

## 入力項目のまとめ

| 画面 | 項目 | 型 | 必須 | バリデーション |
|---|---|---|---|---|
| ウェルカム | - | - | - | - |
| 新規登録 | メールアドレス | Text | ✓ | メールフォーマット |
| 既存ユーザー | メールアドレス | Text | - | メールフォーマット |
| | パスコード | Number | ✓ | 4～6 桁 |
| 組織選択 | 組織 | Select | ✓ | リストから選択 |
| ロール確認 | - | Display | - | - |
| セットアップ | 表示名 | Text | ✓ | 1～50 文字 |
| | プロフィール画像 | File | ✗ | JPG / PNG（2MB以下） |
| | メール通知 | Checkbox | ✗ | - |
| | Slack 連携 | Checkbox | ✗ | - |

---

## フロー完了の判定基準

✅ **オンボーディング完了条件**:
1. メール認証 or パスコード認証 → 完了
2. 組織コンテキスト選択 → 完了（1組織のみなら自動）
3. ロール確認画面を表示 → 確認
4. プロフィール（表示名）入力 → 完了
5. ダッシュボード初期表示 → **完了**

---

## 推奨シーケンス時間（初回ユーザー）

| ステップ | 所要時間 |
|---|---|
| ウェルカム画面 → メール認証 | 30 秒 |
| メール確認・コード入力 | 1 分 |
| 組織選択 | 15 秒 |
| ロール確認 | 30 秒 |
| プロフィール入力 | 1 分 |
| ダッシュボード表示 | 5 秒 |
| **合計** | **約 3 分 30 秒** |

---

## THEBRANCH コンセプト反映

✨ **「翌朝から機能する組織」の実現**:
1. **最小限の情報**: メール + 表示名だけで参加可能
2. **ロール明確化**: 参加直後に自分の権限を理解
3. **即座のアクション**: ダッシュボードで最初のタスクがすぐ見える
4. **親切な UX**: 認証から 3 分 30 秒ですべて完了
