# THEBRANCH オンボーディングUX・フロー設計書

**プロダクトビジョン:** 「翌朝から機能する組織を手にできる世界」

ユーザーがビジョンを入力してから、実際に初タスク実行まで、最短5ステップで到達するフロー設計。

---

## 1. フロー全体像

```
┌─────────────────────────────────────────────────────────────┐
│ THEBRANCH オンボーディングウィザード                      │
└─────────────────────────────────────────────────────────────┘
        ↓
┌─ Step 0: ビジョン入力 ────────────────────────────────┐
│ ユーザーが組織のビジョン・目標をテキスト入力             │
│ 入力例）「営業チーム立ち上げ、月商1000万円達成」        │
└───────────────────────────────────────────────────────┘
        ↓
┌─ Step 1: AI部署自動提案 ───────────────────────────────┐
│ ビジョンを AI が分析 → 最適な部署構成を提案              │
│ - 部署テンプレート（営業・CS・企画等）                   │
│ - メンバー構成・スキル要件                               │
│ - 予算目安・給与体系提案                                 │
│ - 初期タスク・プロセス提案                               │
│ ユーザーが提案を選択 or カスタマイズ                      │
└───────────────────────────────────────────────────────┘
        ↓
┌─ Step 2: 詳細設定 ─────────────────────────────────────┐
│ ユーザーが詳細を入力・確認                               │
│ - 部署名、責任者名、メンバー数                           │
│ - 予算、KPI、優先順位                                   │
│ - テンプレートの同期先（SFA等）                         │
└───────────────────────────────────────────────────────┘
        ↓
┌─ Step 3: 初タスク実行 ──────────────────────────────────┐
│ AI が自動生成した初期タスク実行                          │
│ - タスク内容の確認                                       │
│ - 実行ボタンクリック                                     │
│ - リアルタイムで部署エージェント起動                     │
│ - ダッシュボードへ自動遷移                               │
└───────────────────────────────────────────────────────┘
        ↓
     ✅ 完了
   「翌朝から組織が動き始める」
```

## 2. 各ステップの詳細設計

### Step 0: ビジョン入力

**目的:** ユーザーの組織ビジョン・課題を引き出す

**UI 構成:**
```
┌─────────────────────────────────────────┐
│ THEBRANCH                               │
│ 「翌朝から組織が動く」体験フロー      │
├─────────────────────────────────────────┤
│ Step 1 of 4                             │
│ ════════════════════════ 25%            │
├─────────────────────────────────────────┤
│ あなたの組織ビジョンを入力してください │
│                                         │
│ ┌─────────────────────────────────────┐ │
│ │ 例）営業チーム立ち上げ              │ │
│ │    月商1000万円達成                 │ │
│ │    顧客満足度90%以上               │ │
│ │                                     │ │
│ │ ※ビジョンを入力すると             │ │
│ │   AI が最適な部署構成を自動提案します│ │
│ └─────────────────────────────────────┘ │
│                                         │
│                        [次へ] [キャンセル]│
└─────────────────────────────────────────┘
```

**フロー:**
1. ユーザーがテキスト入力（100〜500字想定）
2. 「次へ」クリック → バリデーション + AI 分析開始（ローディング表示）
3. AI 分析完了後、Step 1 へ遷移

**DB:**
- `onboarding_id`: 一意の ID
- `user_id`: ユーザーID
- `vision_input`: テキスト（最大 500 字）
- `current_step`: 0
- `created_at`, `updated_at`

**UI 構成:**

**ユーザーアクション:**
- [選択] → Step 2 へ

- [スキップ] → カスタムテンプレート作成へ

- `suggested_template`: 推奨テンプレートID

## 4. DB スキーマ

### `user_onboarding_progress` テーブル

```sql
CREATE TABLE user_onboarding_progress (
  onboarding_id TEXT PRIMARY KEY,
  user_id TEXT NOT NULL,
  
  -- Step 0
  vision_input TEXT,
  
  -- Step 1
  suggested_template_id TEXT,
  suggestion_reason TEXT,
  
  -- Step 2
  dept_id TEXT,
  dept_name TEXT,
  manager_name TEXT,
  members_count INTEGER,
  budget INTEGER,
  kpi TEXT,
  integrations JSON,
  
  -- Step 3
  initial_tasks JSON,
  agent_status TEXT,  -- 'created', 'activated', 'running'
  
  -- Metadata
  current_step INTEGER DEFAULT 0,
  completed_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (dept_id) REFERENCES departments(id)
);

CREATE INDEX idx_user_onboarding ON user_onboarding_progress(user_id);
CREATE INDEX idx_current_step ON user_onboarding_progress(current_step);
```
