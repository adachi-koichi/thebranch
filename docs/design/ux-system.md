# THEBRANCH UX デザインシステム

**作成日**: 2026-04-23  
**バージョン**: 1.0  
**フェーズ**: Release 1.0 MVP

---

## 概要

THEBRANCHのUXデザインシステムは、**「ひとりがユニコーンになれる時代へ」** というプロダクトビジョンを実現するために設計された、統合的なデザイン・ユーザー体験の枠組みです。

このシステムは、以下の5つのUXピラーを軸に構成されています：

1. **翌朝から動く** - セットアップからデプロイまで30分以内
2. **専門家の目線** - AIエージェントの推奨・最適化
3. **透明性** - リアルタイムダッシュボード・監視
4. **スケーリング** - 複数部署・複数チームへの拡張
5. **成功実感** - KPI・目標達成の可視化

---

## 1. コアデザイン原則

### 1.1 デザイン哲学

| 原則 | 説明 | 実装例 |
|---|---|---|
| **最小化** | 必須機能のみに絞る | セットアップウィザード: 5ステップ ≤8分 |
| **認知負荷低減** | 複雑な判断を最小化 | テンプレート選択型→自動推奨 |
| **可視化** | すべての状態・進捗を可視化 | KPI ダッシュボード・リアルタイム更新 |
| **即座の効果実感** | ユーザーが価値を即座に体感 | 初日から AI エージェント稼働 |
| **信頼性** | システム透明性・説明責任 | 決定理由・根拠の明示 |

### 1.2 UX層別構成

```
【戦略層】 プロダクトビジョン
  「ひとりがユニコーンになれる時代へ」
         ↓
【体験層】 UXピラー（5つ）
  1. 翌朝から動く
  2. 専門家の目線
  3. 透明性
  4. スケーリング
  5. 成功実感
         ↓
【機能層】 コアフィーチャー（5個）
  1. セットアップウィザード
  2. テンプレート展開・管理
  3. AI Q&A
  4. 監視ログ
  5. KPI・目標管理
         ↓
【実装層】 UI コンポーネント
  - フロントエンド (React/Vue)
  - バックエンド API (Python/FastAPI)
  - データベース (SQLite/KuzuDB)
```

## 2. ユーザーペルソナ別 UX戦略

### 2.1 3つの主要ペルソナ

| ペルソナ | 市場規模 | 主要ペイン | UX優先度 |
|---|---|---|---|
| **P1: 起業家** | 日本 5万人/年 | スピード・スキル | 翌朝から動く / 専門家の目線 |
| **P2: 複業マネジャー** | 日本 10万人 | 時間・判断 | 翌朝から動く / 透明性 |
| **P3: 営業責任者** | 日本 6万人 | 効率・コンプライアンス | スケーリング / 成功実感 |

### 2.2 ペルソナ別ユースケース

**P1: 起業家（営業部を立ち上げた朝）**
```
朝7時: 「営業部を立ち上げたい」→ セットアップウィザード開始
朝7:30: テンプレート選択（営業テンプレート）
朝8:00: AI エージェント自動配置確認
夜中: バックグラウンドでセットアップ完了
翌朝: 営業ダッシュボード・AI エージェント稼働確認

🎯 成功: 実際に営業フローが動き始めた実感
```

**P2: 複業マネジャー（複数部署の進捗確認）**
```
朝9時: ダッシュボードで全部署のKPI確認
朝10時: 問題検出→ AI Q&A で原因分析
朝11時: テンプレート活用で新規施策実行
夜21時: ログで1日の成果確認

🎯 成功: 複数部署を効率的に管理できた
```

**P3: 営業責任者（チームマネジメント）**
```
朝9時: チーム別ダッシュボードで進捗確認
朝10時: 個別メンバーの成績推移・コーチング機会発見
朝11時: テンプレート適用で営業プロセス標準化
夜17時: メンバー評価・人事配置の提案

🎯 成功: チーム全体の生産性向上を実現
```

## 3. コアUIパターン

### 3.1 セットアップウィザード UI

**目標**: 30分以内にセットアップ完了（検証基準: ≤8分）

```
【ステップ1】ビジョン入力
  ├─ 質問: 「何のビジネスをしていますか?」（自由記入）
  ├─ 質問: 「売上目標は?」（数値入力）
  └─ 時間: 2分

【ステップ2】テンプレート選択
  ├─ 提示: 営業 / CS / 企画 / HR等の部署テンプレート
  ├─ 説明付きで選択可能（カード UI）
  └─ 時間: 2分

【ステップ3】初期設定確認
  ├─ AI推奨: 給与 / 採用人数 / KPI 提示
  ├─ ユーザーが確認・調整
  └─ 時間: 2分

【ステップ4】AI エージェント確認
  ├─ 表示: 「SalesBot」「CSBot」等が自動配置
  ├─ 能力・役割説明
  └─ 時間: 1分

【ステップ5】デプロイ確認
  ├─ メッセージ: 「本日夜中にセットアップ完了。明朝をお楽しみに」
  ├─ 完了時刻指定可能
  └─ 時間: 1分

✅ 合計: ≤8分
```

### 3.2 ダッシュボード UI レイアウト

```
【Header】
  ├─ ロゴ / オーガナイゼーション名
  ├─ 部署切り替え
  └─ ユーザー設定 / ログアウト

【Sidebar】
  ├─ ダッシュボード
  ├─ タスク管理
  ├─ 部署管理
  ├─ AI エージェント
  ├─ テンプレートライブラリ
  ├─ 統合・連携
  └─ 設定

【Main Area】
  ├─ Overview
  │  ├─ KPI 概要（4-6個の重要指標）
  │  ├─ リアルタイムチャート
  │  └─ リスク検出アラート
  ├─ Kanban Board
  │  ├─ タスク列（To Do / In Progress / Done）
  │  ├─ 優先度・ラベル表示
  │  └─ ドラッグ&ドロップ対応
  ├─ Analytics
  │  ├─ KPI トレンド分析
  │  ├─ メンバー別成績
  │  └─ 期間比較
  └─ Risk Detection
     ├─ 異常値検出
     ├─ 予測警告
     └─ 推奨アクション

【Footer】
  ├─ ステータス / API 接続状況
  └─ フィードバック / ヘルプ
```

### 3.3 テンプレート管理 UI

```
【テンプレートライブラリ】
  ├─ テンプレートカード一覧
  │  ├─ テンプレート名
  │  ├─ サムネイル
  │  ├─ 説明 (1-2行)
  │  ├─ 採用企業数
  │  ├─ レーティング
  │  └─ [詳細] [適用]ボタン
  ├─ フィルタ
  │  ├─ 業種別
  │  ├─ 規模別
  │  └─ カスタマイズ可否
  └─ [テンプレート作成] ボタン

【テンプレート詳細】
  ├─ 概要
  │  ├─ 説明
  │  ├─ 対象ペルソナ
  │  └─ 推奨企業規模
  ├─ プロセスフロー
  │  └─ Gherkin 形式の自動化フロー表示
  ├─ 初期設定
  │  ├─ AI エージェント配置
  │  ├─ KPI テンプレート
  │  └─ 調整可能項目
  └─ [適用] ボタン
```

## 4. インタラクション パターン

### 4.1 フィードバック・確認

**すべての重要操作には確認・フィードバック**

| 操作 | フィードバック | 時間 |
|---|---|---|
| テンプレート適用 | トースト: 「テンプレート適用中...」 + プログレスバー | 2-5秒 |
| AI Q&A 実行 | 段階的表示: 「分析中...」 → 「結果を出力...」 | 3-10秒 |
| ログ取得 | リアルタイム更新 + 「最新xx件」ラベル | 即座 |
| KPI 更新 | 値変更時にサクセスメッセージ + グラフ更新 | 1秒 |

### 4.2 エラーハンドリング

**ユーザー体験を損なわない エラー対応**

```
【エラーレベル】
  
Critical (利用不可)
  → ユーザーに即座に通知
  → サポート連絡先表示
  → 代替案提示（テンプレート活用等）

Warning (機能制限)
  → トースト警告
  → 代替動作の提示
  
Info (参考情報)
  → 非表示またはツールチップ
  → ユーザーが気になれば確認可能
```

### 4.3 ナビゲーション フロー

```
ログイン → オンボーディング → ダッシュボード
  ↓         ↓                   ↓
  ├─────→ 再ログイン時はスキップ
                ↓
          部署選択 → メインダッシュボード
                      ├─ タスク管理
                      ├─ テンプレート
                      ├─ AI Q&A
                      └─ 設定
```

### 5.2 タイポグラフィ

| 要素 | フォント | サイズ | ウェイト |
|---|---|---|---|
| ページタイトル | 日本語: Hiragino Sans / 英語: Inter | 32px | Bold (700) |
| セクションタイトル | 同上 | 24px | SemiBold (600) |
| 本文 | 日本語: 游ゴシック / 英語: Inter | 14px | Regular (400) |
| サマリー | 同上 | 12px | Regular (400) |
| ボタンテキスト | 同上 | 14px | Medium (500) |

### 5.3 スペーシング

```
基本単位: 8px

【マージン】
  - セクション間: 32px
  - カード間: 16px
  - コンポーネント内: 8-16px

【パディング】
  - ページ: 16px (モバイル) / 24px (デスクトップ)
  - カード: 16px
  - ボタン: 8px (上下) / 16px (左右)
```

## 6. アクセシビリティ

### 6.1 WCAG 2.1 AA 準拠

| 項目 | 要件 | 実装 |
|---|---|---|
| **色のコントラスト比** | 4.5:1 以上 | すべての本文・CTA で検証 |
| **キーボード操作** | Tab / Enter で全機能操作可能 | フォーカス指標明確化 |
| **Alt テキスト** | すべての画像に記述 | 自動画像説明AI活用 |
| **フォーカス順序** | 論理的順序 | DOM順序に一致 |

### 6.2 モバイル対応

| デバイス | 画面幅 | レイアウト |
|---|---|---|
| スマートフォン | 320-480px | シングルカラム + ハンバーガーメニュー |
| タブレット | 600-960px | 2カラム + サイドバー |
| デスクトップ | 1200px+ | 3ペイン（サイド / メイン / サブ） |

## 8. 検証・テスト基準

### 8.1 ユーザー検証（初期リサーチ）

| テスト | 対象 | 基準 |
|---|---|---|
| セットアップ時間 | 5人のP1ユーザー | ≤8分で完了 |
| テンプレート採用率 | 50人のP2ユーザー | ≥80% が推奨テンプレート採用 |
| エラー率 | 1000回のセットアップ | <1% のエラー発生 |
| 満足度 | ユーザーアンケート | ≥4.5/5.0 |

### 8.2 UAT 確認項目

- [ ] セットアップウィザード: 5ステップを8分以内に完了
- [ ] テンプレート適用: ダッシュボード即座表示
- [ ] AI Q&A: 応答時間 < 3秒
- [ ] ダッシュボード: リアルタイム更新
- [ ] エラー画面: 適切なメッセージ・サポート情報表示
