## 2. ユーザージャーニー・フロー図

### 2.1 新規ユーザー（初回アクセス）

```
[訪問者]
   ↓
[/ ランディングページ] ← キャッチコピー・メリット表示
   ↓
[CTA: 「今すぐ始める」] クリック
   ↓
[/signup ウィザード開始]
   ├─→ [Step 1] メール入力 or OAuth
   ├─→ [Step 2] パスワード or OAuth確認
   ├─→ [Step 3] プロフィール入力
   └─→ [送信] メール送信
   ↓
[/verify-email メール認証]
   ├─→ 確認メール受信
   ├─→ リンククリック or コード入力
   └─→ [認証完了]
   ↓
[/onboarding/welcome ウェルカム]
   ├─→ 部署テンプレート選択
   ├─→ メンバー初期設定
   └─→ [完了]
   ↓
[/dashboard メインダッシュボード]
   ├─→ チュートリアル表示（スキップ可能）
   └─→ タスク作成開始
```

### 2.2 既存ユーザー（再訪問・日常利用）

```
[ユーザー]
   ↓
[/login ログインページ]
   ├─→ メール + パスワード or OAuth
   └─→ [ログイン]
   ↓
[/dashboard メインダッシュボード]
   ├─→ 概観 (Overview)
   ├─→ Kanban ボード
   ├─→ リスク検出
   └─→ 分析 (Analytics)
   ↓
[操作選択]
   ├─→ [+ タスク作成] → /create-task
   ├─→ [部署管理] → /departments/:id/settings
   ├─→ [エージェント管理] → /agents
   ├─→ [統合設定] → /integrations
   └─→ [ユーザー設定] → /settings
```

### 2.3 部署設定・カスタマイズフロー

```
[/dashboard] で「部署設定」クリック
   ↓
[/departments/:id/settings]
   ├─ メンバー管理タブ
   │   ├─ メンバー追加 / 編集 / 削除
   │   └─ 権限設定
   │
   ├─ プロファイルタブ
   │   ├─ 部署情報
   │   ├─ アイコン / カラー
   │   └─ Slack / Discord 連携
   │
   ├─ スキルタブ
   │   ├─ 技術スタック
   │   ├─ ドメインナレッジ
   │   └─ カスタムプロンプト
   │
   └─ 権限タブ
       ├─ ロール別権限マトリックス
       ├─ 承認フロー
       └─ 監査ログ
   ↓
[変更を保存]
```

## 3. ナビゲーション構造・階層関係

### 3.1 グローバルナビゲーション（全ページ共通）

```
┌─────────────────────────────────────────────┐
│ 🏢 THEBRANCH  [検索]  🔔  🔧  👤          │
└─────────────────────────────────────────────┘
```

**要素**:
- **ロゴ**: クリック → /dashboard
- **検索**: グローバル検索 (タスク・部署・エージェント検索)
- **通知アイコン**: 新着通知ドロップダウン
- **設定アイコン**: クイック設定メニュー
- **プロフィール**: ユーザーメニュー (プロフィール / ログアウト)

### 3.2 左サイドバーナビゲーション（ダッシュボード内）

```
┌────────────────────┐
│ 📊 ダッシュボード    │
├────────────────────┤
│ 🎯 タスク          │ → /tasks
│ 📂 部署            │ → /departments
│ 🤖 エージェント     │ → /agents
│ 🔗 統合            │ → /integrations
│ 📊 分析            │ → /dashboard?tab=analytics
├────────────────────┤
│ ⚙️ 設定            │ → /settings
│ ❓ ヘルプ          │ → /help
└────────────────────┘
```

**ナビゲーション特性**:
- スティッキー（スクロール時も表示）
- 折りたたみ可能（モバイル）
- アクティブ状態を強調表示
- ホバー時: サブメニュー表示（あれば）

### 3.3 タブナビゲーション（内部ページ）

**部署設定ページ** (/departments/:id/settings):
```
┌─────────────────────────────────────────┐
│ 📋 メンバー | 👤 プロファイル | 💡 スキル | 🔐 権限 │
└─────────────────────────────────────────┘
```

**ダッシュボード** (/dashboard):
```
┌──────────────────────────────────────────┐
│ 📊 Overview | 📈 Kanban | 📉 Analytics | 🚨 Risks │
└──────────────────────────────────────────┘
```

## 5. 認証フロー・セッション管理

### 5.1 ログイン状態の判定

```
[ページアクセス]
   ↓
[JWT トークン確認]
   ├─→ [有効] → コンテンツ表示
   ├─→ [期限切れ] → Refresh Token で更新
   │   ├─ [成功] → 新 Access Token 取得
   │   └─ [失敗] → /login へリダイレクト
   └─→ [無効] → /login へリダイレクト
```

### 5.2 保護ページ（認証必須）

**認証が必須なページ**:
- /dashboard
- /departments/*
- /tasks/*
- /agents/*
- /settings

**認証不要なページ**:
- / (ランディング)
- /signup
- /login
- /verify-email

## 10. アクセシビリティ・SEO考慮事項

### 10.1 アクセシビリティ (WCAG 2.1 AA)

- **キーボード操作**: Tab キーで全操作可能
- **スクリーンリーダー対応**: aria-label, role 属性
- **コントラスト**: 色のみに依存しない（テキスト説明併記）
- **フォーカス表示**: 明確なフォーカスリング
- **エラー表示**: 入力エラー時の説明を画面リーダーで読み上げ可能

### 10.2 SEO

**メタデータ**:
- Title: ページごとに一意 (例: "部署設定 | THEBRANCH")
- Description: ページ内容を簡潔に説明
- OG タグ: 共有時のプレビュー

**ページ構造**:
- H1: ページタイトル（1個のみ）
- H2 以下: 見出し階層の正確な使用
