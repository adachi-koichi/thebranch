# Phase 4: 部署マーケットプレイス UX 設計

## Context（なぜこの設計が必要か）

THEBRANCH は Phase 3（オンボーディングウィザード）で「ビジョン入力→部署提案→初タスク実行」のフローを定義しました。しかし、Phase 3 の Step 1「AI部署自動提案」は以下の課題を持ちます：

1. **テンプレート提案アルゴリズムが未実装** - ビジョン入力後、どのテンプレートを提案するのかの仕組みがない
2. **マッチスコア・推奨度が表示されない** - ユーザーが複数提案の中から最適なものを選べない
3. **テンプレート比較 UI がない** - 複数テンプレート間の違いが可視化されていない
4. **カスタマイズ時のリアルタイム計算がない** - 人数・予算変更による影響が反映されない

Phase 4 マーケットプレイス UX 設計は、これらの課題を解決し、「翌朝から動く」感を実現する詳細な UX を設計します。

## 4. テンプレート提案ロジック設計

### 提案アルゴリズムの3層構造

#### Layer 1: ビジョン解析

**入力**: ユーザーのビジョンテキスト（100-500字）

**処理**:

1. **キーワード抽出**
   - 部署カテゴリーキーワード: 営業、マーケティング、企画、経理、法務、開発、採用 等
   - ビジネス目標キーワード: 売上増加、費用削減、品質向上、スピードアップ、自動化 等
   - エンティティ型キーワード: 金額（「月商1000万」「予算500万」等）、人数（「3-20名」等）、期間（「3ヶ月」「年度末」等）

2. **エンティティ認識**
   - 金額抽出: 正規表現 `\d+万円|\d+円|\$\d+` 等
   - 人数抽出: 正規表現 `\d+名|~\d+人` 等
   - 期間抽出: 「3ヶ月」「年度末」「Q2」 等

3. **意図分類**
   - 新規立ち上げ: 「立ち上げ」「開始」「構築」キーワード含有
   - 既存部署拡大: 「拡大」「強化」「改善」キーワード含有

**出力**: 
```json
{
  "keywords": [
    {"text": "営業", "type": "department", "weight": 1.0},
```

#### Layer 2: スコアリング

**各テンプレートに対して 0-100 のスコアを計算**

スコア計算式:
```
match_score = keyword_match_score(70%) 
            + entity_match_score(20%) 
            + intent_match_score(10%)

keyword_match_score = 
  Σ(extracted_keyword.weight × template_keyword_mapping.weight) / count(extracted_keywords)

entity_match_score = 
  (amount_match_score + headcount_match_score) / 2
  where amount_match_score = テンプレート予算レンジとの適合度 (0-100)
  where headcount_match_score = テンプレートメンバー数レンジとの適合度 (0-100)

intent_match_score = 
  intent_alignment_score (0-100)
  「新規立ち上げ」のテンプレートなら +20 ボーナス
```

**計算例**:

Finance テンプレート（ビジョン: 「決算業務の自動化」）

1. キーワードマッチング:
   - 「決算」: weight 0.95 → 95%
   - 「経理」含意: weight 0.8 → 80%
   - 「自動化」（intent）: weight 0.5 → 50%
   - keyword_match_score = (95 + 80 + 50) / 3 = 75%

2. エンティティマッチング:
   - 金額なし: amount_match = 0%
   - 人数なし: headcount_match = 0%
   - entity_match_score = 0%

3. 意図マッチング:
   - 新規部署立ち上げ: +15
   - intent_match_score = 15%

4. **最終スコア計算**:
   ```
   match_score = 75 × 0.7 + 0 × 0.2 + 15 × 0.1
               = 52.5 + 0 + 1.5
               = 54%
   ```

Sales テンプレート（同じビジョン）

1. キーワードマッチング:
   - 「営業」: weight 1.0 → 100%
   - 「月商」: weight 0.85 → 85%
   - keyword_match_score = (100 + 85) / 2 = 92.5%

2. エンティティマッチング:
   - 金額「月商1000万」: テンプレート予算レンジ ¥150万～¥500万に対して
     → 月額スコア = 250 / 1000 = 25% (高め)
   - entity_match_score = 25%

3. 意図マッチング:
   - 新規部署立ち上げ: +20
   - intent_match_score = 20%

4. **最終スコア計算**:
   ```
   match_score = 92.5 × 0.7 + 25 × 0.2 + 20 × 0.1
               = 64.75 + 5 + 2
               = 71.75% → 72%
   ```

#### Layer 3: ランキング・推奨

**スコア上位 3-5 個をランキング表示**

```
推奨ランキング:
  1. Sales テンプレート: 92% (スコア > 80% → 推奨テンプレート)
  2. Marketing テンプレート: 65% (50-80% → 代替案)
  3. Finance テンプレート: 54% (50-80% → 代替案)
  4. Dev テンプレート: 48% (< 50% → 参考情報)
  5. HR テンプレート: 30% (< 50% → 参考情報)
```

**ランキング構成**:
1. **推奨テンプレート**（スコア ≥ 80%）：大きく強調、選択状態で表示
2. **代替案**（スコア 50-80%）：複数表示、順位表示、選択可能
3. **その他**（スコア < 50%）：参考情報として表示、選択可能だが非推奨
