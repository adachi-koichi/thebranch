## 実行サマリー

### 戦略的な位置づけ

**THEBRANCHの核となる3つのビジョン**

```
プロダクトビジョン（戦略層）
  「ひとりがユニコーンになれる時代へ」
         ↓
UXビジョン（体験層）
  1.翌朝から動く 2.専門家の目線 3.透明性 4.スケーリング 5.成功実感
         ↓
実装戦略（機能層）
  MVP: 5コア機能（ウィザード、テンプレート、AI Q&A、ログ、KPI）
  Phase 2: スケーリング・自動化機能
  Phase 3: 予測・シミュレーション機能
```

## MVP（Release 1.0）実装計画

### MVP コアフィーチャー（5個）

#### Feature 1: セットアップウィザード

**目的**: P1・P2 の「翌朝から動く」を実現

**ユースケース**
- 起業家が、「営業部を立ち上げたい」と思った時点で、30分以内にセットアップ完了

**実装仕様**
```
ステップ 1: ビジョン入力（2分）
  - 「何のビジネスをしていますか?」（自由記入）
  - 「売上目標は?」（予測値）

ステップ 2: テンプレート選択（2分）
  - 「営業」「CS」「企画」等の部署テンプレート提示
  - 説明付きで選択可能

ステップ 3: 初期設定確認（2分）
  - AI による推奨給与・採用人数・KPI の提示
  - ユーザーが確認・調整

ステップ 4: AI エージェント確認（1分）
  - 「SalesBot」「CSBot」等が自動配置されたことを確認

ステップ 5: デプロイ確認（1分）
  - 「本日夜中にセットアップを完了します。明朝をお楽しみに」
  - 完了時刻を指定可能
```

**技術スタック**
- フロントエンド: React / Vue（ステップ型UI）
- バックエンド: Python / FastAPI（テンプレートロジック）
- DB: SQLite（設定保存）+ KuzuDB（関係管理）

**検証基準**
- ✅ 完了時間: ≤8分（P1 5ユーザー）
- ✅ テンプレート採用率: ≥80%
- ✅ エラー率: <1%

---

#### Feature 2: テンプレート展開・管理

**目的**: 「即座に動く」組織体制の実現

**実装仕様**
```
テンプレートタイプ
  - 営業テンプレート: リード発掘 → 初期接触 → クロージング
  - CS テンプレート: 顧客サポート → アップセル → 満足度管理
  - 企画テンプレート: 市場分析 → 施策立案 → 実行管理
  - HR テンプレート: 採用スクリーニング → オンボーディング → マネジメント

テンプレートの含有物
  - プロセスフロー（Gherkin BDD で定義）
  - AI エージェントの指示セット（プロンプト群）
  - 初期 KPI・目標設定
  - ダッシュボードレイアウト
```

**技術スタック**
- Gherkin フォーマット（BDD で部門フロー定義）
- YAML（テンプレート設定）
- KuzuDB（テンプレート間の関係管理）

**検証基準**
- ✅ テンプレート展開時間: <1分
- ✅ デプロイ後の稼働率: ≥99%
- ✅ カスタマイズ可能性: ≥50%（ユーザーが設定変更可能）

---

#### Feature 3: AI コンサルタント（Q&A チャット）

**目的**: P1・P2・P3 の「専門家の目線」を実現

**実装仕様**
```
AI コンサルタント群
  1. Sales Agent
     - リード評価、営業戦略、成約パターン分析
  2. HR Agent
     - 採用スクリーニング、給与設計、人事評価
  3. Finance Agent
     - 予算配分、ROI分析、資金予測
  4. Legal Agent
     - コンプライアンス、契約確認、リスク警告

使用方法
  - チャット形式で質問入力
  - 回答に「根拠」を付記（「業界平均」「過去事例」等）
  - 関連資料・テンプレートの提示

応答時間
  - 目標: <30秒
  - テンプレート回答: <3秒
  - LLM利用時: <15秒
```

**技術スタック**
- Claude API / Anthropic SDK（LLM統合）
- PromptTemplate（各エージェント別テンプレート）
- VectorDB（事例検索・類似回答検出）

**検証基準**
- ✅ 応答時間: <30秒（平均）
- ✅ 回答満足度: ≥4.5/5.0
- ✅ 採択率: ≥60%（提案が実装される割合）

---

#### Feature 4: 実行ログ・監査ダッシュボード

**目的**: P3 の「透明性」を実現

**実装仕様**
```
記録内容
  1. AI による実行ログ
     - 「SalesBot が 14:30 に Lead Scoring 実行」
     - 実行対象：顧客ID, スコア値, 根拠ルール
  
  2. 判定根拠の明示
     - 「なぜこのリードを HIGH 評価したか」
     - 根拠：「過去同業種で 80% 成約」
  
  3. ユーザーアクション
     - 「マネジャーが営業提案を却下」
     - 理由：「顧客ブランド的にリスク」

検索・分析機能
  - タイムスタンプ検索
  - AI エージェント別フィルタ
  - 根拠ルール別検索
  - エクスポート機能（監査用）
```

**技術スタック**
- SQLite（ログテーブル）
- JSON（判定根拠の構造化）
- Elasticsearch（全文検索）

**検証基準**
- ✅ ログ記録率: ≥99%
- ✅ 検索応答時間: <2秒
- ✅ 監査対応度: ≥95%

#### Feature 5: KPI ダッシュボード

**目的**: すべてのペルソナの「成功実感」を実現

**実装仕様**
```
ダッシュボード構成

【メインビュー】
  - リアルタイム KPI 概観
    - 営業: 受注数、成約率、パイプライン
    - CS: 顧客満足度、チャーン率、NPS
    - Finance: 売上、コスト、利益率

【週次サマリー】
  - 「今週のハイライト」
    - 成約 +5件、売上 +¥500k, NPS +8pt
  - 注目指標（異常値警告）

【比較ビュー】
  - 先月比、YoY比、目標比

【AIインサイト】
  - 「成約率が前週比 -10% です。リード質が低下している可能性」
  - 提案: 「リード発掘ルールを見直しましょう」

【アラート】
  - KPI 未達時に自動通知
  - 「本月末まで XX 件必要です」
```

**技術スタック**
- React / Vue（リアルタイムUI）
- D3.js / Plotly（グラフ）
- WebSocket（リアルタイム更新）
- Claude API（AI インサイト生成）

**検証基準**
- ✅ 更新レイテンシー: <10秒
- ✅ グラフ表示速度: <3秒
- ✅ ユーザー確認率: ≥80%（毎日ダッシュボード開く）

```

各質問ごとに記録
- 回答時間
- 回答の満足度（5段階）
- 根拠の明示度
- 採択意思（この提案を実装するか）

期待結果
- 全質問の平均回答時間: ≤30秒
- 平均満足度: ≥4.0/5.0
- 根拠明示率: ≥90%
```

## リスク管理

### 技術的リスク

| リスク | 対策 |
|---|---|
| AI Q&A の誤回答 | テンプレート回答で 90% をカバー。LLM は事例ベース・ファクトベース |
| 監査ログの記録漏れ | 各 AI アクション前に ログ記録を確認。テストカバレッジ ≥95% |
| スケーラビリティ | 初期は SQLite + KuzuDB。月 10万 DAU 超過時に DB マイグレーション計画 |

### ビジネスリスク

| リスク | 対策 |
|---|---|
| ユーザー獲得遅延 | ProductHunt / Y Combinator / 起業家コミュニティを活用 |
| 機能要件変更 | Phase 1 は 5 機能に限定。追加要件は Phase 2+ へ |
| コンペティター | 透明性（監査ログ）と スピード（テンプレート）で差別化 |
