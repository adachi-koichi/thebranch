## ペルソナ 1: 野心的起業家 - 機能要件

### 利用シーン
**起業直後（Week 1）から部署拡大（6ヶ月）まで、継続的に**「営業立ち上げ」「採用」「成果確認」を支援

### 機能要件群

#### 機能 1-1: セットアップウィザード
**優先度**: 🔴 MVP 必須  
**利用シーン**: アカウント作成直後（初回のみ）  
**ユーザーストーリー**:  
```
As a 起業家
I want 「営業立ち上げテンプレート」を選択するだけで、
So that 翌朝には営業チームが稼働できる状態になる
```

**画面構成**:
1. **ウィザード Step 1: 部署タイプ選択**
   - 営業・マーケティング / CS・サクセス / 企画・運営 / カスタム
   - 各テンプレートのプレビュー（メンバー構成、給与目安、SFA連携）
   - 目標: ≤2分で選択

2. **ウィザード Step 2: 基本情報入力**
   - 部署名、責任者名、予算上限
   - メンバー数目安（「営業 3人」など）
   - 目標売上・KPI（「月商 XXX万円」など）
   - バリデーション: 入力ミスの即座指摘
   - 目標: ≤3分で入力

3. **ウィザード Step 3: 確認・承認**
   - AI による設定提案の表示
     - 給与体系提案（市場相場に基づく）
     - SFA設定・連携（Google Sheets / Salesforce / カスタム）
     - 営業プロセス設定（リード発掘 → 初期接触 → クロージング）
   - 「推奨に従う」 vs 「カスタマイズ」の選択肢
   - 目標: ≤2分で承認

**機能詳細**:
- **市場相場データベース**: 業界・職種・地域別の給与相場を内蔵
- **テンプレート展開**: 選択したテンプレートに基づき、AI が自動で初期設定
- **リアルタイム検証**: 「この予算では、営業 3人 + 1マネジャーで月給与 XXX万円。着手可能か?」と即座に警告
- **進捗表示**: ステップ 1/3 → 2/3 → 3/3 の視覚的進捗表示

**受け入れ基準**:
- ウィザード完了時間: ≤8分
- 各ステップ滞留時間: ≤2分/ステップ
- テンプレート採用率: ≥80%
- 初回セットアップ成功率: ≥95%

#### 機能 1-2: AI コンサルタント Q&A
**優先度**: 🔴 MVP 必須  
**利用シーン**: 日常業務中（毎日 1-5件）  
**ユーザーストーリー**:
```
As a 起業家
I want 「給与相場は?」「このコスト構造、リスク?」と聞くと
So that 30秒で根拠付きの回答を得られて、意思決定に自信を持てる
```

**画面構成**:
1. **Q&A インターフェース**
   - チャット形式のQ&A（自然言語対応）
   - 質問例の提示（「営業 3人の給与予算は?」「採用コスト相場は?」）
   - 入力フィールド（テキスト）
   - 送信ボタン

2. **AI 回答表示**
   - 直接回答（「営業 3人の月給与予算: 180-240万円」）
   - 根拠表示（「市場相場: XX万円 / 人。あなたの予算上限と比較して XX%」）
   - リスク警告（「注意: その給与では優秀層を採用しにくい」）
   - 関連リンク（「参考: 給与体系設計ガイド」）

3. **質問履歴・保存**
   - 過去の Q&A を「チーム資料」として保存可能
   - 「このアドバイスに従った」「従わなかった」の結果記録

**機能詳細**:
- **マルチドメイン対応**: HR（給与・採用）, Finance（予算・キャッシュフロー）, Legal（コンプライアンス）, Operations（オペレーション設計）
- **コンテキスト認識**: ユーザーの既入力情報（部署名、予算、メンバー数）を踏まえた回答
- **信頼度表示**: 回答の信頼度を 3段階（高 / 中 / 低）で表示
- **参考データ**: 業界平均、成功事例、失敗事例を併記

**受け入れ基準**:
- Q&A 利用頻度: 週 5-10件
- AI 回答満足度スコア: ≥4.0 / 5.0
- 「AI 提案を実行した」の採択率: ≥70%
- 平均応答時間: ≤2秒

#### 機能 1-3: 採用管理・AI スクリーニング
**優先度**: 🔴 MVP 必須  
**利用シーン**: 採用時（月 2-5件）  
**ユーザーストーリー**:
```
As a 起業家
I want 応募者を AI が自動スクリーニング & 初期面接を実施
So that 採用時間を 3-4週間 → 1-2週間に短縮できて、採用コストを 80% 削減
```

**画面構成**:
1. **採用管理ダッシュボード**
   - 「新しい募集を開始」ボタン
   - 募集一覧（タイトル、ステータス、応募数、スクリーニング済み数）
   - 募集詳細ビュー

2. **募集フォーム**
   - 募集職種（ドロップダウン）
   - 必須スキル（複数選択タグ）
   - 給与範囲
   - 雇用形態（正社員 / 契約 / パート）
   - 募集締切

3. **応募者管理**
   - 応募者リスト（名前、スクリーニング結果、進捗ステータス）
   - スクリーニング結果の詳細表示
     - AI 評価: 「スキルマッチ度」「経験度」「文化フィット度」
     - 評価根拠: 「このスキルは募集要件と XX% マッチ」
     - 推奨アクション: 「初期面接へ進める」 vs 「見送り」
   - フィルタ・ソート: スクリーニング結果に基づくフィルタリング

4. **初期面接 AI**
   - 初期面接の日時設定（自動スケジューリング）
   - AI が自動で初期面接を実施（テキストベースの質問応答）
   - 面接結果の自動レポート（「このエンジニアは、Python・JavaScript に強く、大規模プロジェクト経験あり」）

**機能詳細**:
- **スキルマッチング**: 応募者の経歴・スキルと募集要件を自動分析
- **カルチャーフィット度**: 企業文化との適合度を評価
- **初期面接 AI**: 標準的な質問（経験、モチベーション、給与期待）を自動実施
- **評価根拠の記録**: AI がなぜこう評価したのかを明示

**受け入れ基準**:
- 採用時間: 3-4週間 → 10日以内
- AI スクリーニング精度: ≥85%（実採用に至った人材の AI 評価 vs 実際のパフォーマンスの相関度）
- 採用コスト/人: 150-200万円 → 30-50万円

#### 機能 1-4: 営業成果ダッシュボード
**優先度**: 🟠 MVP後期  
**利用シーン**: 毎日夕方（1-2回）  
**ユーザーストーリー**:
```
As a 起業家
I want 営業チームの「成約数」「パイプライン」「売上予測」を一目で確認
So that リアルタイムで営業進捗を把握して、意思決定できる
```

**画面構成**:
1. **KPI カード** （上部）
   - 本日成約数 / 週間成約数 / 月間売上見込み
   - メンバー別の成約件数ランキング
   - リアルタイム更新（SFA / Salesforce との連携）

2. **パイプラインビュー**
   - Kanban ボード形式（リード発掘 → 初期接触 → 提案 → クロージング → 成約）
   - 各段階の案件数と金額
   - ドラッグ&ドロップで進捗更新

3. **売上予測チャート**
   - 折れ線グラフ: 実績 vs 予測
   - 月間売上見込み（確度別）
   - 警告アラート（「目標達成リスク: 現在の進捗では、月間売上目標の 70% に留まる見込み」）

**機能詳細**:
- **SFA 連携**: Salesforce / HubSpot / Google Sheets との自動同期
- **AI 予測**: 過去成約パターンに基づく売上予測
- **異常検知**: 「営業 A の成約率が通常の 50% に低下」など、パフォーマンス異常を自動検知

**受け入れ基準**:
- ダッシュボード読み込み時間: ≤2秒
- データ更新遅延: ≤30分
- 売上予測の精度: ±10%

#### 機能 1-5: スケール時の自動対応
**優先度**: 🟠 Phase 2  
**利用シーン**: 営業 3人 → 5人への拡大時  
**ユーザーストーリー**:
```
As a 起業家（3ヶ月目）
I want 営業チームが 3人 → 5人に拡大しても、プロセスが自動スケール
So that 管理オーバーヘッドなしに、チーム規模を拡大できる

**機能詳細**:
- **テンプレート再展開**: 追加メンバーのオンボーディングが自動化
- **プロセス自動スケール**: リード数が増えても、AI が自動で初期接触・追跡を実施
- **給与・予算自動計算**: 「営業 5人での給与予算」「予想月商」を自動計算

**受け入れ基準**:
- オンボーディング時間: ≤1日
- スケール後の成約率: 維持

## ペルソナ 2: 複業マネジャー - 機能要件

### 利用シーン
**副業 5-10時間/週 の時間制約下で、「ルーチン自動化」「意思決定支援」「チーム心理安全性」を実現**

### 機能要件群

#### 機能 2-1: 給与・勤怠・経費自動処理
**優先度**: 🔴 MVP 必須  
**利用シーン**: 月初・月末（毎月 2回）  
**ユーザーストーリー**:
```
As a 複業マネジャー
I want 給与計算・勤怠確認・経費精算を完全自動化
So that 月末の手作業（現在 80分）を 10分以下に削減して、実ビジネス時間を増やせる
```

**画面構成**:
1. **給与管理ダッシュボード**
   - メンバー一覧（名前、職種、給与、勤続月数）
   - 給与計算ステータス（「計算済み」「承認待ち」「支払い完了」）
   - 「給与計算を実行」ボタン（1クリック）

2. **勤怠管理**
   - メンバーの勤怠入力（自動記録 or 手動修正）
   - 勤務時間サマリー（「今月: 60時間」）
   - 異常検知（「メンバー A の勤務時間が通常の 50% に低下」）

3. **経費精算管理**
   - メンバーからの経費申請一覧
   - 「承認」 / 「修正依頼」ボタン
   - 月別経費サマリー

4. **給与・経費の自動計算結果**
   - 「総給与: XXX万円」「経費: YYY万円」「手取り: ZZZ万円」
   - 予算との比較（「予算 300万円 vs 実績 280万円」）

**機能詳細**:
- **自動給与計算**: 時給 / 月給 / 成果給 の自動計算
- **税金・社会保険**: 自動控除と計算
- **銀行連携**: 給与自動振込（オプション）
- **勤怠自動記録**: Slack / Google Calendar との連携

**受け入れ基準**:
- 給与計算時間: 現在 30分 → ≤5分
- エラー率: ≤1%
- 「完全自動で十分」の満足度: ≥80%

As a 複業マネジャー

   - 最近のタスク進捗

**受け入れ基準**:
- アプリ起動時間: ≤3秒
- 意思決定AI回答表示: ≤10秒
- ユーザー満足度: ≥4.0/5.0

#### 機能 2-3: AI 意思決定支援
**優先度**: 🔴 MVP 必須  
**利用シーン**: 日常判断時（週 3-5件）  
**ユーザーストーリー**:
```
As a 複業マネジャー
I want 「営業成約率 20%、改善すべき?」と聞くと
So that 業界平均との比較と改善施策を 30秒で得て、自信を持って意思決定できる
```

**画面構成**:
1. **意思決定 Q&A インターフェース**
   - 質問入力フィールド
   - 質問例の提示（「このツール導入は ROI に見合う?」「メンバーの給与は適切?」）

2. **AI による構造化回答**
   ```
   Q: 「営業成約率 20%、改善すべき?」
   
   A:
   ┌─ 評価
   │  業界平均: 35%
   │  あなた: 20%
   │  → ランク: 下位 30% （改善の余地あり）
   │
   ├─ 改善施策
   │  1. 初期接触の効率化（リード時間 30% 削減）
   │  2. クロージング支援 AI 導入（成約率 +5-10%）
   │  3. 営業研修（新人向け、2週間）
   │
   ├─ 実装コスト
   │  1. リード自動化ツール: 月 5万円
   │  2. AI 導入: 月 10万円
   │  3. 研修: 初回 20万円
   │  → 合計: 初月 35万円、月間 15万円
   │
   └─ ROI 予測
      月商 1000万円 × 成約率向上 +5% = +50万円/月売上
      3ヶ月で投資回収可能
   ```

3. **判断履歴の保存**
   - 「この提案を実行した」「実行しなかった」を記録
   - 実行後の結果追跡（「予測: +50万円 → 実績: +45万円」）

**機能詳細**:
- **多次元分析**: 業界平均・競合・カテゴリー別での比較
- **改善施策の優先度付け**: 実装難易度 / 期待効果 / 時間軸での提案
- **実装コスト推定**: 導入コスト、月間ランニングコスト、ROI を自動計算

**受け入れ基準**:
- AI 回答提示時間: ≤2秒
- 回答の構造化度: ≥90%（「評価」「施策」「コスト」「ROI」すべて含む）
- 提案採択率: ≥70%

As a メンバー（複業プロジェクト）

So that 仕事がスムーズに進んで、プロジェクトへの満足度が上がる
```

**画面構成**:

**受け入れ基準**:
- メンバー満足度: ≥75%
- マネジャーへの報告削減: 50%

| **メンバー満足度** | 60% | ≥85% | AI サポート + メンバーの自動回答 |
| **マネジャーの心理的負荷** | 高 | 低 | AI による相談対応で時間短縮 |

## ペルソナ 3: 組織管理者 - 機能要件

### 利用シーン
**営業部門 20-50人の効率化・コンプライアンス・人間判断維持を支援**

### 機能要件群

#### 機能 3-1: 営業プロセス標準化 & AI 効率化
**優先度**: 🔴 MVP 必須  
**利用シーン**: 営業管理時（継続）  
**ユーザーストーリー**:
```
As a 営業責任者
I want トップセールスの成功パターンを AI が自動化して営業全体に展開
So that 新人営業でも 30% → 50% の成約率を実現できる
```

**画面構成**:
1. **営業プロセス管理ダッシュボード**
   - 営業ステップの可視化（リード発掘 → 初期接触 → 提案 → クロージング）
   - 各ステップの成功率（トップセールス: 80%, 平均: 30%, 新人: 10%）
   - ステップボトルネックの自動検出（「初期接触成功率が低い」）

2. **トップセールス分析**
   - トップセールスの成功パターン抽出
   - 「初期メール文言」「クロージング時期」「価格提示タイミング」などの分析
   - 成功パターンを営業全体に標準化するボタン

3. **営業プロセス自動実行**
   - AI が初期メール・フォローアップを自動実行
   - 営業は「提案・クロージング」に注力
   - AI と営業の役割分担を明確化

**機能詳細**:
- **成功パターン抽出**: 機械学習で成約した案件の共通パターンを自動抽出
- **初期メール自動生成**: トップセールスの文言をベースに、顧客別にカスタマイズして自動生成
- **フォローアップスケジューリング**: 営業が「フォローアップが必要」と判定すると、AI が自動で 1週間後にメール送信
- **営業支援 AI**: 営業が「次どうするか?」を聞くと、AI が過去データから最適な次ステップを提案

**受け入れ基準**:
- 営業効率改善: 現在 基準値 → ≥50% 改善
- 成約率: 30% → ≥45%
- リード発掘時間削減: 営業時間の 40% → 15% 以下
- 営業満足度: ≥80%

#### 機能 3-2: リード自動発掘
**優先度**: 🔴 MVP 必須  
**利用シーン**: 毎日稼働（自動）  
**ユーザーストーリー**:
```
As a 営業責任者
I want AI が「見込み顧客 500社」を毎日自動抽出
So that 営業は「営業活動」に集中して、成約数を増やせる
```

**画面構成**:
1. **リード発掘ダッシュボード**
   - 本日新規リード数（「本日: 50社」）
   - リード源（「LinkedIn: 30社」「業界データベース: 20社」）
   - リード品質スコア（「高確度リード: 15社」「中確度: 35社」）

2. **リード一覧**
   - 企業名、代表名、業界、従業員数、成約確度
   - 「営業 A に割り当て」「提案メール送信」などのアクション

3. **リード発掘設定**
   - 「業界」「従業員規模」「購買予算」などの条件設定
   - AI が自動で条件に合致したリードを発掘

**機能詳細**:
- **複数データソース連携**: LinkedIn, 帝国データバンク, 業界 DB, 過去成約データなど
- **AI 品質スコアリング**: 「この企業は購買可能性 XX%」を自動判定
- **重複排除**: 既出顧客・成約済み顧客を自動除外
- **自動割り当て**: 営業の過去成功パターンに基づいて、最適な営業に自動割り当て

**受け入れ基準**:
- 日次リード数: ≥50社
- リード品質スコア精度: ≥80%（実際の成約確度との相関）
- 営業の「この条件は使える」の評価: ≥90%

#### 機能 3-3: 監査ログ完全記録 & 透明性確保
**優先度**: 🔴 MVP 必須  
**利用シーン**: コンプライアンス監視（定期監査時）  
**ユーザーストーリー**:
```
As a コンプライアンス担当者
I want AI の判定根拠が完全に記録されて検索可能である
So that 「なぜこの営業判定?」が常に説明できて、監査に対応できる
```

**画面構成**:
1. **監査ログダッシュボード**
   - 「AI が判定した案件」「営業が最終判定した案件」の集計
   - コンプライアンス違反の有無

2. **ログ詳細検索**
   - 営業名、案件名、判定日時で検索
   - 検索結果に「AI 判定根拠」を表示

3. **根拠の可視化**
   ```
   案件名: "ABC社 成約判定"
   AI 判定: 成約確度 95%
   判定根拠:
   - 顧客データ: 過去成約企業と 95% マッチ
   - 購買パターン: 過去 6ヶ月で業界内に導入事例 3件
   - 営業進捗: 初回接触から 30日（平均成約期間 35日）
   使用データ: "顧客DB_2026-04-22", "業界ニュース_2026-03-15"
   ```

4. **アラート & 警告**
   - 古いデータを使用した判定の警告（「この判定は 3ヶ月前のデータ。データ更新を推奨」）
   - 異常判定の検出（「この営業の判定が通常と異なる」）

**機能詳細**:
- **判定ログ記録**: AI が判定するたびに、判定根拠・使用データ・実施時刻をすべて記録
- **検索・監査**: 監査担当者が任意の時期・営業の判定を遡及検索
- **バージョン管理**: AI モデルの更新履歴を記録（「2026-04-01 版」「2026-04-15 版」）
- **排出**: Excel / CSV で監査レポート自動生成

**受け入れ基準**:
- ログ記録率: 100%
- ログ検索応答時間: ≤1秒
- 監査時の「根拠説明可能率」: ≥95%
- コンプライアンス違反: 0件/月

#### 機能 3-4: コンプライアンスルール設定
**優先度**: 🟠 MVP後期  
**利用シーン**: 管理時（ポリシー設定）  
**ユーザーストーリー**:

```

**画面構成**:
1. **ルール設定管理**
   - 「新しいルールを追加」ボタン
   - ルール一覧（ルール名、条件、アクション）

2. **ルール作成フォーム**
   ```

   条件: 契約金額 ≥ 100万円
   アクション: 営業部長の承認が必須

   - 「ルール適用 → 承認 / 却下」の履歴

**機能詳細**:

- **段階的承認**: 「部長承認 → 役員承認」の複数段階ルール設定可能
- **例外管理**: 「既存顧客は金額下限を下げる」などのルール例外設定可能

**受け入れ基準**:
- ルール設定の容易度: ≥4.0/5.0
- ルール適用率: ≥99%
