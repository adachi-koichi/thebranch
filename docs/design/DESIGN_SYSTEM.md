## 1. デザイン原則

### 1.1 コア原則

1. **シンプリシティ** (Simplicity)
   - UI は必要最小限に（過度な装飾なし）
   - ユーザーの認知負荷を低減
   - 直感的な操作フロー

2. **一貫性** (Consistency)
   - 全ページで統一されたビジュアル言語
   - 同じ機能は同じビジュアルで表現
   - コンポーネントの再利用

3. **アクセシビリティ** (Accessibility)
   - WCAG 2.1 AA 準拠
   - カラーのみに依存しない設計
   - キーボード操作・スクリーンリーダー対応

4. **スケーラビリティ** (Scalability)
   - 複数部署・大量タスクに対応可能
   - パフォーマンス重視
   - レスポンシブ設計

5. **信頼性** (Trust)
   - 透明な情報開示
   - エラーハンドリング明確化
   - セキュリティ表示

## 8. トランジション & モーション

### 8.1 トランジション時間

```
┌──────────────────────────────────────┐
│ Duration Scale                       │
├──────────────────────────────────────┤
│ ultra-fast: 50ms  (micro interactions)
│ fast:      100ms  (hover effects)    │
│ base:      200ms  (standard)         │
│ slow:      300ms  (modal, page)      │
│ slower:    500ms  (complex anim)     │
└──────────────────────────────────────┘

用途:
- 50ms:   ボタンクリック、コピーボタン
- 100ms:  ホバー、フォーカス変更
- 200ms:  配色変更、シャドウ変更
- 300ms:  モーダル表示/閉鎖、ページ遷移
- 500ms:  複雑なアニメーション、ローディング
```

### 8.2 イージング関数

```
ease-in:       cubic-bezier(0.4, 0, 1, 1)      開始遅い
ease-out:      cubic-bezier(0, 0, 0.2, 1)      終了遅い
ease-in-out:   cubic-bezier(0.4, 0, 0.2, 1)    両端遅い
ease-linear:   cubic-bezier(0, 0, 1, 1)        一定速度
ease-bounce:   custom bounce effect             バウンス効果
```

### 8.3 モーション例

```
┌────────────────────────────────────┐
│ Button Hover                       │
```

## 9. ボーダー & ラディウス

### 9.1 Border Radius Scale

```
┌──────────────────────────────────┐
│ Radius Scale                     │
├──────────────────────────────────┤
│ radius-xs:    2px   (small)      │
│ radius-sm:    4px   (buttons)    │
│ radius-md:    6px   (cards)      │
│ radius-lg:    8px   (containers) │
│ radius-xl:   12px   (modals)     │
│ radius-2xl:  16px   (hero)       │
│ radius-full: 9999px (pills/circles)
└──────────────────────────────────┘

用途:
- 2px:    バッジ、小要素
- 4px:    ボタン、小カード
- 6px:    入力フィールド
- 8px:    カード、パネル
- 12px:   モーダル
- 16px:   ヒーローセクション
- full:   アバター、プロフィール画像
```

```
エラー:       2px solid error-500
```
