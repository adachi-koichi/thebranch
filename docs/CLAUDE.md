# docs/ — ルールと構成

このディレクトリはTHEBRANCHの全仕様書・設計書を管理するObsidianボルト兼ドキュメントルート。

## 仕様書を探すとき

**まず `SPEC_INDEX.md` を読むこと。**
カテゴリ別の全ファイル一覧と「何を調べるか → どこを見るか」のクイックリファレンスがある。
目的のドキュメントが見つかったらパスを確認して直接読む。

## フォルダ構成

```
docs/
├── SPEC_INDEX.md       # AIエージェント用インデックス（必読）
├── index.base          # Obsidian UI用ビュー定義
├── design/             # 設計書（UX・アーキテクチャ・DB・API設計）
├── api/                # API仕様書
├── product/            # プロダクトビジョン・機能仕様
├── test_reports/       # テスト結果レポート
└── *.md                # システム仕様・フロー設計（ルート直置き）
```

## ドキュメント追加・変更のルール

1. **既存ファイルを上書きする前に内容を確認する** — 同名ファイルが存在する場合はdiffを確認し、新旧どちらが正しいかユーザーに確認してから上書きする
2. **新規ドキュメントを追加したら `SPEC_INDEX.md` を更新する** — ファイルパス・タイトル・内容の概要を該当カテゴリに追記する
3. **配置先フォルダ**:
   - UX/UI/アーキテクチャ/DB/API/セキュリティの設計 → `design/`
   - APIエンドポイント仕様（OpenAPI相当） → `api/`
   - プロダクトビジョン・機能仕様・ユーザーリサーチ → `product/`
   - テスト結果レポート → `test_reports/`
   - フロー設計・データモデル・システムアーキテクチャ（横断的なもの） → ルート

## 触ってはいけないもの

- `.obsidian/` — Obsidianの設定ディレクトリ。編集禁止
- `.trash/` — Obsidianのゴミ箱。操作不要
- `CLAUDE.md` 自体 — ユーザーの指示がある場合のみ編集する

## Obsidianとの共存

- このディレクトリはObsidianボルトとして使われている
- `index.base` はObsidian Bases（UI表示定義）。YAMLフォーマットで編集可能
- フロントマター（YAMLヘッダ）を追加することでBasesのフィルタリングが強化される
