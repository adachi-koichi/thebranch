> 「組織は、生まれながらの特権ではない。アイデアを持つすべての個人が翌朝から機能する組織を手にできる世界。採用でも外注でもなく、部署ごと、AIで動かす。ひとりが、ユニコーンになれる時代へ。」

## コア機能一覧（7つのモジュール）

### 1. **組織構造テンプレートエンジン**

ユーザーが業種・段階別のテンプレートを選択し、AI がカスタマイズして「翌朝から動く組織」を構築。

**主要ユースケース**
- スタートアップ初期段階（1-3名）向け「最小構成」テンプレート
- 成長期（5-10名）向け「部署分化」テンプレート
- スケール期（20-50名）向け「職能別」テンプレート

**含まれる要素**
- 組織図（部署・ロール定義）
- 給与体系（職能別給与相場・昇進基準）
- 稟議フロー（支出レベル別の承認プロセス）
- 勤怠・シフト管理ルール

---

### 2. **採用・オンボーディング自動化**

候補者スクリーニングから初期育成まで、AI エージェントが採用プロセスを自動化。

**主要ユースケース**
- 候補者スクリーニング（書類選考、スキル評価）
- 初期面接のスケジューリング・リマインダー
- 面接フィードバック集約・フィードバック通知
- オンボーディングチェックリスト自動化
- 初期育成支援（チームへのアサイン調整）

**含まれる要素**
- JD（職務記述書）テンプレート
- 面接質問データベース（ロール別）
- オンボーディングチェックリスト
- 新入社員向けマイクロラーニング

---

### 3. **AI 意思決定支援エージェント**

HR・財務・法務の簡単な質問に対し、即座に回答・提案するコンサルタント型 AI。

**主要ユースケース**
- 「このメンバーの給与は適切か?」→ 市場相場データに基づく提案
- 「経費削減の余地はあるか?」→ カテゴリ別削減提案
- 「新規採用のタイミングは?」→ 予算・組織規模に基づく推奨
- 「退職者が出た場合の対応は?」→ 法務・HR リスク告知
- 「この契約条件は妥当?」→ 業界標準値との比較

**含まれる要素**
- 市場データベース（給与・外注費相場）
- リスク判定エンジン
- テンプレート提案システム

---

### 4. **統合ダッシュボード**

給与、予算、メンバー情報、進捗管理が 1 つの画面で可視化。

**主要ユースケース**
- **経営ダッシュボード**: 月次売上、経費、キャッシュフロー、チーム規模推移
- **HR ダッシュボード**: メンバー情報、給与・評価・離職率、採用パイプライン
- **プロジェクトダッシュボード**: タスク進捗、マイルストーン、リソース配置
- **アラート**: 予算超過、採用遅延、離職率上昇時の自動通知

**含まれる要素**
- リアルタイム更新（連携システムから自動同期）
- カスタマイズ可能なウィジェット
- レポート生成（月次・四半期・年次）
- 予測分析（3ヶ月先予算予測など）

---

### 5. **ルーチンワーク自動化**

シフト管理、経費精算、定例ミーティング招集など、定型業務を自動実行。

**主要ユースケース**
- 勤怠管理（Slack から打刻、自動記録）
- 経費精算（レシート画像 AI 判定、自動仕訳）
- 定例ミーティング自動招集（カレンダー統合）
- 給与計算補助（タイムシートから自動計算）
- 在庫・備品管理（定期発注自動化）

**含まれる要素**
- Slack インテグレーション
- OCR（画像認識）
- カレンダー連携
- 会計システム連携

---

### 6. **モバイルアプリ（スマートフォン対応）**

出先・移動中での意思決定・承認機能。

**主要ユースケース**
- Slack を通じた承認フロー（支出承認、休暇申請承認）
- 簡易ダッシュボード（KPI 確認）
- 意思決定支援 AI とのチャット
- オンボーディングチェックリスト確認
- リアルタイムアラート（予算超過、採用候補者到着など）

**含まれる要素**
- Web アプリ + ネイティブアプリ（iOS/Android）
- Slack Bot
- Push 通知

---

### 7. **チームコーチング・フィードバック自動化**

AI が中立的なコーチ・相談役になり、チーム内の心理安全性を向上。

**主要ユースケース**
- 定期フィードバック自動化（1on1 質問テンプレート、回答集約）
- パルスサーベイ（チーム満足度の定期測定）
- 心理安全性スコア算出・改善提案
- メンタルヘルス相談窓口（AI チャット）
- チームビルディング提案（イベント提案、アイスブレーカーなど）

**含まれる要素**
- 質問テンプレートライブラリ
- 感情分析エンジン
- フィードバック集約・可視化

## 詳細仕様：ユーザーストーリー & 受け入れ条件

### **機能1: 組織構造テンプレートエンジン**

#### US-001: スタートアップ初期段階の組織テンプレート選択

**ユーザーストーリー**
```
前提条件: ユーザーは新規スタートアップの創業者で、組織構造を決めたい（1-3名段階）

As a 野心的起業家
I want 業種別の「最小構成」組織テンプレートを選択し、AI がカスタマイズできる
So that 1 週間以内に「給与体系・稟議フロー」が決まり、組織構造が整備される
```

**受け入れ条件**
- [ ] テンプレート選択画面が 3 個以上のテンプレート（SaaS, マーケティング, コンサル等）を提供
- [ ] 選択後、AI が「あなたの組織に最適な」給与体系・稟議フロー案を生成（5分以内）
- [ ] ユーザーが AI 提案を確認・編集でき、「確定」ボタンで即座に反映
- [ ] 生成された組織図が JSON/CSV で出力可能（スプレッドシート等への連携）

#### US-002: 成長期の組織テンプレート適用

**ユーザーストーリー**
```
前提条件: 初期テンプレートから 6 ヶ月経過し、5-10 名に拡大したユーザー

As a 野心的起業家
I want 現在の組織構造から「成長期」テンプレートへ移行でき、新規に追加すべき部署・ロールを提案してほしい
So that 組織の拡大に伴う混乱を最小化し、スムーズにスケールできる
```

**受け入れ条件**
- [ ] 現在の組織人数・部署構成を入力すると、推奨される次段階テンプレートが表示される
- [ ] 「どのロールを追加すべきか」という提案が、給与予算と共に提示される
- [ ] テンプレート移行時に「新規メンバーのオンボーディングチェックリスト」が自動生成される

### **機能2: 採用・オンボーディング自動化**

#### US-003: JD 自動生成 & 採用フロー開始

**ユーザーストーリー**
```
前提条件: ユーザーが「営業職 1 名を採用したい」と判断した

As a 野心的起業家
I want 「職種・必要スキル・給与」を入力すると、AI が JD を自動生成し、採用フローが開始できる
So that 採用プロセスの準備に 1 日以上の時間を短縮できる
```

**受け入れ条件**
- [ ] 「新規採用」ボタンで職種・スキル要件入力画面が表示される
- [ ] 入力から 5 分で JD（職務記述書）が自動生成される
- [ ] 生成 JD は「スキル要件・給与・勤務地・評価基準」を含む
- [ ] JD 確定後、LinkedIn/Indeed 等への自動掲載が可能（外部 API 連携）

#### US-004: 候補者スクリーニング自動化

**ユーザーストーリー**
```
前提条件: JD が確定し、候補者応募が 30 名集まった

As a 野心的起業家
I want 30 人の応募から「スキル・経験」ベースの自動スクリーニングが行われ、上位 5-10 名の候補者リストが自動生成される
So that スクリーニング業務に 10 時間以上の時間を削減でき、面接準備に集中できる
```

**受け入れ条件**
- [ ] 応募者の履歴書/職務経歴書が AI で読み込まれ、スコアリングされる
- [ ] 「スキルマッチ度」「経験度」「カルチャーマッチ」3 項目でランク付けされる
- [ ] スコアリング結果が表で表示（ユーザーが手動で上下可能）
- [ ] 面接招集候補者が「ワンクリック」で決定でき、自動でメール送信

#### US-005: 面接スケジューリング自動化

**ユーザーストーリー**
```
前提条件: スクリーニング完了し、候補者 5 名を面接招集する

As a 野心的起業家
I want 候補者に面接日時を提案し、自動でカレンダー調整・リマインダーメール送信が実行される
So that 面接スケジューリングの手動調整（メール往復）が 0 になる
```

**受け入れ条件**
- [ ] 「面接招集」ボタンで候補者・面接官のカレンダーが表示される
- [ ] AI が「3 候補のスケジュール両立可能な日時」を自動提案する
- [ ] 候補者に「提案日時」がメール送信され、確定時に自動でカレンダーに登録される
- [ ] 面接 1 日前に候補者・面接官に自動リマインダーが送信される

#### US-006: オンボーディングチェックリスト自動生成

**ユーザーストーリー**
```
前提条件: 新入社員の入社日が決定した

As a 採用担当者（複業マネジャー）
I want 新入社員のロール・部署に基づいて、オンボーディングチェックリストが自動生成される
So that 新入社員のオンボーディング漏れを防ぎ、初期育成を効率化できる
```

**受け入れ条件**
- [ ] 新入社員情報（氏名・部署・ロール・入社日）を入力すると、チェックリストが自動生成される
- [ ] チェックリストは「事前準備（IT アカウント、デスク準備等）」「1 日目」「1 週間」「1 ヶ月」に分類
- [ ] 各チェック項目は責任者が割り当てられ、期日までに完了が求められる
- [ ] 未完了項目は自動でリマインダーが送信される

### **機能3: AI 意思決定支援エージェント**

#### US-007: 給与相場提案

**ユーザーストーリー**
```
前提条件: ユーザーが新規採用メンバーの給与を決める段階

As a 野心的起業家
I want 「職種・経験年数・地域」を入力すると、市場相場に基づいた給与提案が 30 秒で表示される
So that 「適切な給与が分からない」という不安を解消し、自信を持って給与を決定できる
```

**受け入れ条件**
- [ ] 「給与相場を知りたい」ボタンで職種・経験年数・地域の入力フォームが表示される
- [ ] 入力後 30 秒以内に「業界平均」「25パーセンタイル」「75パーセンタイル」が表示される
- [ ] 「この給与で採用できる確率は?」という成功確度が表示される（市場データに基づく）

#### US-008: 経費削減提案

**ユーザーストーリー**
```
前提条件: ユーザーが経費削減を検討している

As a 複業マネジャー
I want 月次経費データを入力すると、「削減可能な領域」が自動提案される
So that 限られた時間で経費を最適化でき、キャッシュフローの改善ができる
```

**受け入れ条件**
- [ ] 月次経費カテゴリ（ツール利用料、外注費、ワークスペース費等）が入力できる
- [ ] 各カテゴリの「業界標準値」との比較が表示される
- [ ] 「削減提案」が「削減額」「リスク」とともに表示される（例: ツール統合で月 5 万削減、ただしオンボーディング 1 日コスト）

#### US-009: 採用タイミング推奨

**ユーザーストーリー**
```
前提条件: ユーザーが「次のロール追加のタイミング」を判断したい

As a 野心的起業家
I want 現在の売上・経費・組織規模を基に、「いつこのロールを採用すべきか」が提案される
So that スケーリングの意思決定が迅速になり、採用タイミングの後手を避けられる
```

**受け入れ条件**
- [ ] 「採用タイミング判定」機能で、現在の経営数字（売上、経費、現在人数等）を入力できる
- [ ] AI が「このロール採用のベストタイミングは〇月」と提案する（予測分析）
- [ ] 根拠として「売上成長率」「経費圧力」「業界成長段階」が表示される

## 機能優先順位（MoSCoW 分析）

### **M（Must Have）- MVP フェーズ 1（3ヶ月）**
必ず含める機能。これなしではプロダクトとして成立しない。

| 優先度 | 機能 | 理由 |
|---|---|---|
| 1 | 組織構造テンプレートエンジン | プロダクトビジョン実現の核（「翌朝から動く組織」） |
| 2 | 採用・オンボーディング自動化 | 全ペルソナの最優先課題（採用スピード） |
| 3 | AI 意思決定支援エージェント | ペルソナ 1/2 が「判断の自信」を求めている |
| 4 | 統合ダッシュボード | 複数ツール統合を実現（情報散在を解消） |
| 5 | ルーチンワーク自動化（Slack 勤怠） | 「限られた時間で最大成果」（複業マネジャーの課題） |

### **S（Should Have）- フェーズ 2（3-6ヶ月）**
できれば含めたい。MVP 後の初期ユーザー拡大に重要。

| 優先度 | 機能 | 理由 |
|---|---|---|
| 6 | モバイルアプリ | 複業マネジャー・企業内リーダーの「外出先対応」要件 |
| 7 | ルーチンワーク自動化（経費精算） | ペルソナ 2 の「時間効率」要件 |
| 8 | チームコーチング・フィードバック | ペルソナ 2/3 の「心理安全性」要件 |

### **C（Could Have）- フェーズ 3+（6ヶ月以降）**
実装できれば高価値だが、MVP では不要。

| 優先度 | 機能 | 理由 |
|---|---|---|
| 9 | 既存システム統合（ERP、給与計算システム等） | ペルソナ 3（企業内リーダー）のスケーリング要件だが、初期ユーザーは少ない |
| 10 | 高度な予測分析（売上予測、採用予測等） | データが必要（ユーザー利用 3ヶ月以降が現実的） |
| 11 | AI による人事評価補助 | 市場で感度が高いが、デリケートさから MVP では避ける |

### **W（Won't Have）- スコープ外**
スコープ外として明示的に除外。

| 項目 | 理由 |
|---|---|
| 給与計算・年末調整の完全代行 | 税務申告との紐付けが必要で、法人責任が重い |
| リモートワーク勤怠管理（生体認証等） | プライバシー課題が大きく、初期段階では不要 |
| AI による採用候補者スコアリング（スキル以外） | 差別リスクが高い（年齢、性別等の推定） |
| Zoom/Meet 等での面接自動評価 | プライバシー・法的課題が大きい |
