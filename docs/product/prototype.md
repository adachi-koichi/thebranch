## プロトタイプ概要

THEBRANCH のMVP段階（フェーズ1）向けプロトタイプは、**「翌朝から動く組織」をノーコードで構築できる** ユーザーインターフェースを実現する。

### 設計方針

1. **シンプルさ**: 3ステップのオンボーディングで組織立ち上げ完了
2. **テンプレート駆動**: 業種別テンプレートで初期セットアップを高速化
3. **透明性**: すべてのAI判断が追跡可能・監査可能
4. **段階的複雑化**: MVP では基本機能に絞り、拡張は段階的に

## ユーザーフロー

### フロー1: 初回ユーザー（組織立ち上げ）

```
[1] 登録・ログイン
    ↓
[2] オンボーディング開始
    - サービスビジョン入力
    - 対象顧客の定義
    ↓
[3] 部署構成選択
    - プリセット: 「営業 + CS」「営業 + 企画」など
    - カスタム: 自由に部署を追加
    ↓
[4] ワークフロー選択
    - 営業フロー: リード獲得 → 顧客化 → LTV管理
    - CS フロー: チケット → 解決 → 満足度
    - 企画フロー: アイデア → 案件化 → 実行
    ↓
[5] AI エージェント配置 & 初期指示
    - 各部署にエージェントを自動配置
    - 初期目標・KPIを設定
    ↓
[6] オンボーディング完了 → ダッシュボード表示
```

### フロー2: 既存ユーザー（日次利用）

```
[1] ログイン
    ↓
[2] ダッシュボード表示
    - 組織全体の状況（KPI, 売上, チーム規模）
    - アラート（予算超過、採用遅延、AI失敗）
    - 今日のTODO（承認待ち、重要タスク）
    ↓
[3] アクション選択
    - 承認・却下（支出、休暇申請）
    - 意思決定支援AIに質問（給与相場、経費削減等）
    - チーム管理（新規採用、オンボーディング）
    ↓
[4] アナリティクス確認
    - 月次レポート（売上、経費、チーム成長）
    - AI パフォーマンス分析
```

## スクリーン設計

### 画面1: オンボーディング（ステップ1/3）

**タイトル**: "あなたのビジョンを教えてください"

**要素**:
- ヒーロータイトル: "翌朝から動く組織を作る"
- 入力フォーム（3項目）:
  1. **サービス名** (テキスト入力)
     例: "AIコンサルティング"
  2. **ビジョンステートメント** (テキストエリア)
     プレースホルダー: "企業の意思決定を透明化し、データドリブンな判断を実現する"
  3. **対象顧客** (セレクトボックス)
     オプション: スタートアップ / 中小企業 / 大企業 / その他
- CTA: "次へ" ボタン（活性化条件: 3項目入力済み）
- サブナビ: "ステップ 1/3"

**デザイン特性**:
- 背景: グラデーション (濃紺 → 深青)
- フォント: 大きく、明確 (ヘッダー: 32px, 本文: 16px)
- 入力フィールド: クリーンな白背景、丸い角
- CTA: プライマリカラー（青系）、ホバーでスケール/シャドウ

---

### 画面2: 部署構成選択（ステップ2/3）

**タイトル**: "組織構成を選んでください"

**要素**:
- **プリセット部署構成** (複数選択可):
  1. "営業 + カスタマーサポート"
     説明: "新規営業を自動化し、顧客対応も自動化"
  2. "営業 + 企画 + CS"
     説明: "フルスタック組織構成"
  3. "企画のみ"
     説明: "アイデアから実行までをAIで自動化"
  4. "カスタム（部署を手動追加）"
     説明: "自由に部署を組み合わせる"

- 選択されたプリセットの場合: 対応するテンプレートが自動提案される
- カスタムの場合: "部署を追加" ボタンで動的フォーム

**デザイン特性**:
- カード形式（3x1 グリッド）
- 各カードはホバーで浮き上がる（elevation）
- 選択状態: チェックアイコン + 背景色変更
- 説明テキスト: グレー、小さいサイズ（14px）

---

### 画面3: ワークフロー選択（ステップ3/3）

**タイトル**: "組織の働き方を定義してください"

**要素**:
- **選択済み部署ごとに推奨ワークフローが表示**:
  
  営業部の場合:
  - "営業フロー（標準）": リード獲得 → 提案 → 受注 → LTV管理
    説明: "B2B営業の典型フロー"
  - "営業フロー（ハイタッチ）": 人間が関わる提案段階
    説明: "高額商材向け"
  
  CS部の場合:
  - "CS フロー（自動化）": チケット自動作成 → AI解決 → 満足度調査
  - "CS フロー（ハイブリッド）": 複雑な案件は人間エスカレーション

- テンプレートプレビュー: 選択時に「フロー図」を表示
  （フロー図は簡潔: ボックス + 矢印、3-5ステップ）

**デザイン特性**:
- テンプレートカード（各部署ごと、セクション化）
- プレビュー: カード下部または右側にフロー図
- カラーコード: 各部署ごとに異なる色（営業=青、CS=緑、企画=紫）

## デザイン原則

### 1. シンプリシティ（Simplicity）
- ボタン数を最小化（メインアクションは1つ）
- 入力フィールドは最小限（3項目以下が目安）
- 初期表示時のコンテンツを絞る（詳細は折りたたみ）

### 2. スキャナビリティ（Scannability）
- 視覚階層を明確に（ヘッダー > サブ > 本文）
- アイコンを効果的に使う（文字だけでなく視覚情報）
- ホワイトスペースを十分に確保

### 3. トランスペアレンシー（Transparency）
- AI判定の根拠を常に表示（「なぜこの判定？」に答える）
- 監査ログは簡単にアクセス可能
- エラーメッセージは親切に（ユーザーが何を直すべきか明確）

### 4. アクセシビリティ（Accessibility）
- コントラスト比 WCAG AA 以上
- キーボードナビゲーション対応
- スクリーンリーダー対応（alt テキスト等）
