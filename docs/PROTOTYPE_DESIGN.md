> 「組織は、生まれながらの特権ではない。アイデアを持つすべての個人が翌朝から機能する組織を手にできる世界。採用でも外注でもなく、部署ごと、AIで動かす。ひとりが、ユニコーンになれる時代へ。」

このドキュメントは、WF#13 Step1-4 の成果物（ビジョン、ユーザーリサーチ、ペルソナ、機能仕様、詳細UI設計）を統合し、実装フェーズへの完全なハンドオフを実現することを目的とします。

## 2. 主要画面ワイヤーフレーム（統合版）

### 2.1 アーキテクチャレイアウト

THEBRANCH は以下の 3 層から構成される：

```
┌─────────────────────────────────────────────────────────┐
│ Layer 1: プリフロー（未認証）                             │
│ ├─ ランディングページ /                                   │
│ ├─ サインアップ /signup                                  │
│ └─ ログイン /login                                       │
├─────────────────────────────────────────────────────────┤
│ Layer 2: オンボーディング（初回セットアップ）              │
│ ├─ プロフィール入力 /onboard/profile                    │
│ ├─ 部署テンプレート選択 /onboard/template              │
│ ├─ 部署メンバー初期設定 /onboard/members               │
│ └─ 完了画面 /onboard/complete                          │
├─────────────────────────────────────────────────────────┤
│ Layer 3: コアプロダクト（運用）                           │
│ ├─ ダッシュボード /dashboard                            │
│ ├─ AI部署管理 /departments                              │
│ ├─ タスク管理 /tasks                                     │
│ ├─ チーム管理 /team                                      │
│ ├─ 意思決定支援 /advisor                                │
│ └─ 設定 /settings                                       │
└─────────────────────────────────────────────────────────┘
```

## 3. コア機能フロー

### 3.1 AIエージェント起動フロー

```
┌─────────────────────────────────────────────────────────┐
│ ユーザー: 部署テンプレート選択 /onboard/template        │
│ 「成長期型（5-10名用）」を選択                           │
└────────────────────┬──────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────┐
│ System: テンプレートメタデータ読み込み                   │
│ - 部署構成: Engineering, Sales, HR/Finance              │
│ - AI エージェント役割定義: TL, Engineer, SM, PM, HR, CFO │
│ - タスク定義: 組織化・採用・財務設定...                  │
└────────────────────┬──────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────┐
│ System: AI エージェント インスタンス作成                  │
│ - Orchestrator がエージェント起動スクリプトを実行        │
│ - 各エージェントに role / persona / initial_task を付与  │
│                                                         │
│ 起動されるエージェント:                                  │
│ 1. Engineering TL (ccc-engineering-manager)            │
│ 2. Sales Manager (ccc-ai-agent)                        │
│ 3. CFO Bot (ccc-ai-agent)                              │
│ 4. HR Manager (ccc-ai-agent)                           │
└────────────────────┬──────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────┐
│ System: 初期タスク生成                                   │
│ - テンプレートに紐づくタスク定義からインスタンスを生成   │
│ - 依存関係を自動解析（DAG）                             │
│ - 各エージェントに担当タスク割り当て                      │
│                                                         │
│ 生成タスク例:                                            │
```
