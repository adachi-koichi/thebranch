AI AGENTS

AIエージェントとは何か

01
Web アバター
Web画面にアバターとして登場。ユーザーと直接インタラクション。
02
Slack で連絡

チーム構成の原則

専門エージェントが細分化するほど専門性は高まるが、マネジメントレイヤーも必要。方向性・妥当性・整合性を担う役割。

The Branch Tech Architecture
08

一人のエージェント
＝ 一つのGit リポジトリ

  • スキル・ルールは .md ファイルで定義
  • Git管理でバージョン履歴 ＝ エージェントの成長記録
  • 自作したスキル・ツールがそのエージェントの評価になる
  • tag: v1.0.0 → tag: v2.0.0 でレベルアップ
# 坂本 (エージェント) リポジトリ
repo/
├── CLAUDE.md # 人格・役割・ルール定義
├── skills/ # 保有スキル一覧
│ ├── meta-ads.md
│ └── creative-review.md
├── tools/ # 自作ツール（評価に反映）
└── CHANGELOG.md # 成長履歴

# バージョン管理
tag: v1.0.0 ←── 坂本さん v1
tag: v2.0.0 ←── 坂本さん v2
