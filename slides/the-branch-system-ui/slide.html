The Branch Agent Definition

エージェント = Git リポジトリ

# エージェント定義例: sakamoto-v2 # repo: the-branch/agents/sakamoto-v2 ──────────────────────────────────── CLAUDE.md # 基本ルール・ペルソナ定義 skills/ ├─ meta-ads.md # Meta広告運用スキル ├─ copy-write.md # コピーライティング └─ reporting.md # レポーティング tools/ ├─ meta-api.py # Meta API ツール └─ slack-bot.py # Slack 通知 CHANGELOG.md # 成長記録（コミットログ） ──────────────────────────────────── # バージョンアップ = スキル追加・改善 $ git tag v2.0.0 -m "ROAS改善スキル追加" $ git clone . ../sakamoto-v2-b-corp/ # → 機密除去 → B社向けにインスタンス化完了
スキル定義
全て .md ファイルで記述
ルール・制約・得意業務を Markdown で定義。LLM が読み込み、即座に行動規範に反映。エンジニア不要で業務チームが編集できる。
バージョン管理
Git コミット = 成長記録
コミットログが学習履歴。タグでバージョン管理。差分で「何が改善されたか」を可視化。育成の ROI を数値で証明できる。
インスタンス化
git clone でコピー展開
機密除去 → リネーム → 即起動。同一エージェントを複数社に同時展開。これがコスト構造を根本から変える。

The Branch Launch Flow

アサインから稼働まで、最短3ステップ

01
カタログ選択
業務に合うエージェントをカタログで選ぶ。スキル・バージョン・価格・評価スコアを確認してチームに追加。
02
チーム編成
複数エージェントをチームとしてグループ化。部署・案件ごとに異なるチームを並行して組める。
03
ルール設定
優先ルール・制約・エージェント名をカスタマイズ。機密保護スコープを定義する。ここが差別化ポイント。
ここが差別化
04
即日稼働
Slack / ブラウザ UI から指示するだけ。マネージャーエージェントが整合性を担保して並行稼働する。

11
The Branch Check Agent

24/7稼働するセキュリティエージェント

01
常時監視
全エージェントの出力をリアルタイムスキャン。機密ワード・不審な外部通信・異常な API コール量を自動検出し、即座にログに記録する。
02
自動アラート
異常検知時は Slack・メールに即通知。重大度レベル別にエスカレーションルートを設定。エージェントを一時停止する権限も持つ。
03
人間は最終判断のみ
日常監視は Check Agent が全自動。人間は「承認 / 却下 / エスカレーション」のみ担う。運用コストを最小化しながら最高水準のガバナンスを維持。

The Branch Integrations

主要 API と接続済み

Meta Ads API
稼働中
広告配信・予算最適化・ROAS 計測。広告代理店フェーズの中心スキル。ビーチヘッド市場で即日活用できる。
Google Ads API
稼働中
キーワード入札・品質スコア改善・レポート自動生成。Meta と組み合わせて横断運用が可能。
Slack
稼働中
指示・承認フロー・アラートを Slack で完結。非エンジニアも操作できるため導入ハードルが低い。
GitHub / Git
稼働中
エージェントリポジトリ管理・バージョン管理・差分追跡の基盤。全エージェント定義を Git で管理。
Claude API
実装中
Anthropic Claude を LLM ランタイムとして使用。長いコンテキスト・拡張思考機能を最大活用する。
Custom Webhook
計画中
任意の外部サービスと接続するカスタム Webhook エンドポイント。クライアント固有の要件に対応。
